# Video Codec & Format Reference

미디어아트 설치 콘텐츠의 코덱/포맷 선택 가이드.

## Codec Comparison

| 코덱 | 컨테이너 | 화질 | 디코딩 부하 | 파일 크기 | 미디어서버 호환 |
|------|----------|------|-----------|----------|---------------|
| HAP | .mov | 중 | **매우 낮음** (GPU) | 매우 큼 | Resolume, TD |
| HAP Q | .mov | 상 | **매우 낮음** (GPU) | 매우 큼 | Resolume, TD |
| NotchLC | .mov | 상 | **매우 낮음** (GPU) | 큼 | disguise, Notch |
| ProRes 422 | .mov | 상 | 중 | 큼 | 범용 |
| ProRes 4444 | .mov | 최상 | 중 | 매우 큼 | 범용 (알파 지원) |
| H.264 | .mp4/.mov | 중-상 | 높음 (CPU) | 작음 | 범용 |
| H.265/HEVC | .mp4/.mov | 상 | 매우 높음 | 매우 작음 | 제한적 |
| DXV | .mov | 중 | **매우 낮음** (GPU) | 큼 | Resolume 전용 |

## 용도별 추천

### 실시간 재생 (미디어서버)
- **1순위**: HAP / HAP Q — GPU 디코딩, CPU 부하 최소
- **2순위**: NotchLC — disguise 환경
- **주의**: H.264/H.265는 CPU 부하가 높아 다채널 재생 시 드롭

### 아카이브 / 마스터
- **ProRes 4444** — 최고 화질, 알파 채널 지원
- **ProRes 422 HQ** — 고화질, 합리적 파일 크기

### 배포 / 프리뷰
- **H.264** (.mp4) — 최소 파일 크기, 범용 재생

### LED 전용
- **HAP Q** 또는 **ProRes 422** — 대부분의 LED 프로세서 호환

|-----|------|
