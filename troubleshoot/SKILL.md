# /troubleshoot — 트러블슈팅 가이드

> **Role**: Technical Director / Site Engineer
> **Trigger**: User invokes `/troubleshoot` or asks about 트러블슈팅, 문제 해결, 장애 대응
> **Output**: `{프로젝트 폴더}/TROUBLESHOOT.docx` + `TROUBLESHOOT.md`

## Purpose

미디어아트 설치 현장에서 발생하는 일반적인 문제의 진단-해결 가이드를 생성하거나, 특정 문제를 진단한다. 비전문가 현장 스태프도 따라할 수 있는 수준으로 작성한다.

## Behavior

### 0. Save Location

> "문서를 어디에 저장할까요?"

### 1. Context Check

`{프로젝트 폴더}`에서 확인:
- `TECH-SPEC.md` — 시스템 구성 파악
- `INSTALL-GUIDE.md` — 정상 상태 기준
- `SITE-SURVEY.md` — 현장 조건

### 2. Mode Selection

**Q1 — 모드**
> "트러블슈팅 가이드를 만들까요, 아니면 지금 발생한 문제를 진단할까요?"

| 모드 | 설명 |
|------|------|
| 가이드 생성 | 프로젝트 시스템 기반 예상 문제 목록 + 대응법 문서 |
| 실시간 진단 | 지금 발생한 특정 증상을 인터랙티브하게 진단 |

### 3A. Guide Generation Mode (프로젝트 타입별 분기)

TECH-SPEC.md의 시스템 구성을 기반으로 예상 문제 목록을 생성한다.

**프로젝트 타입별 추가 트러블슈팅 항목:**

| 프로젝트 타입 | 타입 고유 문제 |
|-------------|-------------|
| 프로젝션 매핑 | 블렌딩 어긋남 (진동/열팽창), 밝기 불균일, 곡면 워핑 틀어짐, 주변광 간섭 |
| LED 디스플레이 | 데드 픽셀/라인, 모아레 패턴, 캐비닛 간 색 불일치, 과열 셧다운 |
| 인터랙티브 | 센서 오인식/미인식, 레이턴시 과다 (>100ms), 다중 사용자 충돌, 환경광 간섭 |
| 키네틱 | 모터 과열/소음, 탈조(step loss), 안전 정지 오작동, 진동/공진 |
| 이머시브 | 다면 이음새 불일치, 음향 반사/울림, 관객 위치별 왜곡, 환기 소음 |

각 타입별 문제는 아래 공통 카테고리 테이블 **이후에** 별도 섹션으로 추가한다:

**디스플레이 문제**
| 증상 | 가능한 원인 | 대응 |
|------|-----------|------|
| 화면 없음 | 케이블 접촉 불량 / 입력 소스 설정 / 신호 없음 | 케이블 재연결 → 입력 소스 확인 → 미디어서버 확인 |
| 화면 깜박임 | 케이블 불량 / 해상도 불일치 / 주사율 불일치 | 케이블 교체 → 해상도 확인 → 주사율 통일 |
| 색상 이상 | 색공간 불일치 / 케이블 종류 오류 / 감마 설정 | 색공간 확인 → HDMI/DP 확인 → 감마 리셋 |
| 프로젝터 블렌딩 어긋남 | 진동 / 열팽창 / 소프트웨어 크래시 | 재캘리브레이션 → 마운트 확인 → 소프트웨어 재시작 |

**네트워크 문제**
| 증상 | 가능한 원인 | 대응 |
|------|-----------|------|
| 장비 간 통신 불가 | IP 충돌 / 서브넷 불일치 / 케이블 | ping 테스트 → IP 확인 → 케이블 교체 |
| OSC/ArtNet 수신 불가 | 포트 설정 / 방화벽 / 네트워크 대역 | 포트 확인 → 방화벽 해제 → 대역 확인 |

**콘텐츠 문제**
| 증상 | 가능한 원인 | 대응 |
|------|-----------|------|
| 프레임 드롭 | 코덱 부적합 / 스토리지 속도 / CPU/GPU 과부하 | 코덱 변환 → SSD 확인 → 성능 모니터 |
| 재생 시작 안됨 | 파일 경로 / 코덱 미지원 / 라이선스 | 경로 확인 → 코덱 확인 → 라이선스 확인 |

**전원 문제**
| 증상 | 가능한 원인 | 대응 |
|------|-----------|------|
| 차단기 트립 | 과부하 / 합선 / 접지 불량 | 부하 분산 → 케이블 점검 → 접지 확인 |
| 장비 미작동 | 콘센트 불량 / 케이블 / 장비 고장 | 다른 콘센트 → 케이블 교체 → 대체 장비 |

**센서 문제**
| 증상 | 가능한 원인 | 대응 |
|------|-----------|------|
| 인식 불가 | USB 연결 / 드라이버 / 범위 밖 | 재연결 → 드라이버 확인 → 위치 조정 |
| 오인식/노이즈 | 환경광 / 반사 / 간섭 | 환경 차단 → 임계값 조정 → 필터 적용 |

### 3B. Live Diagnosis Mode

> "어떤 증상이 나타나고 있나요?"

사용자의 증상 설명을 받고, 하나씩 진단 질문을 던진다:
1. 증상이 언제부터 발생했는지 (갑자기 / 점진적 / 처음부터)
2. 변경한 것이 있는지 (소프트웨어 업데이트, 케이블 이동, 장비 추가)
3. 영향 범위 (전체 / 특정 장비만)

진단 트리를 따라가며 가능한 원인을 좁힌다.

### 5. Post-generation

```
트러블슈팅 가이드가 생성되었습니다.
항목: 디스플레이 N개, 네트워크 N개, 콘텐츠 N개, 전원 N개, 센서 N개

다음 단계:
- /maintenance 로 정기 점검 일정을 만들어 문제를 사전 예방할 수 있습니다
```

### 6. Feedback Loop (피드백 기록)

실시간 진단 모드(3B)에서 문제를 해결한 후, 해결 결과를 `{프로젝트 폴더}/FIELD-LOG.md`에 기록하도록 안내한다:

> "이 문제의 해결 결과를 FIELD-LOG에 기록할까요? (향후 유사 프로젝트에 참고됩니다)"

기록 시:
- 증상, 원인, 해결 방법을 FIELD-LOG.md의 해당 섹션에 추가
- 기존 트러블슈팅 가이드에 없었던 새로운 패턴이면 TROUBLESHOOT.md에도 항목 추가를 제안
- FIELD-LOG.md가 없으면 `references/FIELD-LOG-TEMPLATE.md` 기반으로 생성

## Core Principles

1. **비전문가도 따라할 수 있게** — 전문 TD가 아닌 현장 스태프가 사용하는 문서.
2. **증상 → 원인 → 대응** 순서 — 원인부터 나열하지 않는다.
3. **간단한 것부터** — 복잡한 원인보다 케이블 접촉 불량 같은 단순 원인을 먼저 확인.
4. **피드백 기록** — 해결된 문제를 FIELD-LOG에 기록하여 references 정확도를 높인다.
5. **한 번에 하나의 질문**
6. **언어를 따른다**
