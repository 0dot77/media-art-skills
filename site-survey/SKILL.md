# /site-survey — 현장 답사 체크리스트/보고서

> **Role**: Site Engineer
> **Trigger**: User invokes `/site-survey` or asks about 현장 조사, 답사, site survey
> **Output**: `{프로젝트 폴더}/SITE-SURVEY.docx` + `SITE-SURVEY.md`

## Purpose

미디어아트 설치 현장의 조사 체크리스트를 생성하거나, 현장에서 수집한 데이터를 구조화된 보고서로 정리한다. 공간 치수, 전력, 네트워크, 환경, 접근성 등을 체계적으로 기록한다.

## Behavior

### 0. Save Location

> "문서를 어디에 저장할까요? (폴더 경로를 지정해주세요)"

### 1. Context Check

`{프로젝트 폴더}`에서 다음 파일을 확인한다:
- `PROJECT.md` — 프로젝트 유형 (디스플레이 유형에 따라 체크 항목이 달라짐)
- `TECH-SPEC.md` — 장비 스펙 (전력 요구량, 네트워크 요구 등)
- `SITE-SURVEY.md` — 이미 있으면 업데이트할지 질문

### 2. Input Gathering (한 번에 하나씩)

**Q1 — 조사 단계**
> "현장 조사 단계를 알려주세요."

| 단계 | 설명 | 출력 |
|------|------|------|
| 방문 전 (Pre-visit) | 현장 가기 전 준비 | 체크리스트 + 준비물 목록 |
| 현장에서 (On-site) | 현장에서 측정값 입력 | 인터랙티브 Q&A → 보고서 |
| 방문 후 (Post-visit) | 수집한 데이터 정리 | 완성 보고서 |

**Q2 — 프로젝트 유형** (PROJECT.md 없을 때만)
> "어떤 유형의 설치인가요?"

유형에 따라 체크 항목이 **크게 달라진다**. 아래 타입별 추가 체크 항목을 적용한다:

| 프로젝트 타입 | 필수 추가 체크 항목 |
|-------------|-----------------|
| 프로젝션 매핑 | 투사 거리, 주변광 레벨(lux), 벽면 색상(Munsell/RAL)/재질/반사율, 프로젝터 마운트 위치 후보, 차광 가능 여부 |
| LED 디스플레이 | 구조물 하중 한계(kg/m²), 가용 전력 용량(A), 전용 회로 여부, LED 프로세서 설치 공간, 환기/냉방 |
| 인터랙티브 | 센서 설치 가능 위치, 센서 FOV 장애물, 관객 동선 폭/방향, 바닥 재질(반사/흡수), 주변 IR 소스(햇빛/히터) |
| 키네틱 | 천장 리깅 포인트 위치/수량, 각 포인트 하중 한계(kg), 안전 거리(관객-기구부), 비상 정지 접근성 |
| 이머시브 | 바닥/벽/천장 전면 치수, 음향 반사/흡수 특성, 출입구 차광, 관객 수용 인원, 비상 대피 경로 |

### 3. Mode-specific Behavior

#### Pre-visit Mode (방문 전)

프린트 가능한 체크리스트를 생성:

**측정 항목 체크리스트**
- [ ] 공간 치수 (가로, 세로, 천장 높이)
- [ ] 기둥/장애물 위치
- [ ] 바닥 재질/내하중
- [ ] 프로젝터 설치 가능 위치 ~ 벽면 거리
- [ ] 천장 리깅 포인트/하중 한계

**전기 체크리스트**
- [ ] 차단기 위치/용량
- [ ] 콘센트 위치/수량
- [ ] 접지 상태
- [ ] 임시 전력 배치 가능 여부

**네트워크 체크리스트**
- [ ] 유선 LAN 포트 위치/수량
- [ ] WiFi 사용 가능 여부/SSID
- [ ] 전용 회선 요청 가능 여부

**환경 체크리스트**
- [ ] 주변광 (암실/자연광/조명)
- [ ] 온습도
- [ ] 소음 수준
- [ ] 환기 상태

**접근 체크리스트**
- [ ] 로딩독/반입 경로
- [ ] 엘리베이터 크기
- [ ] 출입문 크기
- [ ] 작업 가능 시간
- [ ] 주차

**촬영 목록**
- [ ] 전체 공간 전경 (4방향)
- [ ] 천장 상세
- [ ] 차단기/콘센트
- [ ] 반입 경로
- [ ] 문/엘리베이터

**준비물 목록**
- 레이저 거리 측정기
- 조도계 (또는 스마트폰 앱)
- 줄자 (백업)
- 카메라
- 메모장/태블릿

#### On-site Mode (현장에서)

한 번에 하나씩 측정값을 질문하며 인터랙티브하게 데이터를 수집한다:

> "천장 높이를 측정해주세요 (m)"
> "가용 전기 용량은? (차단기 확인)"
> "프로젝터 설치 위치에서 벽까지 거리는?"

각 답변을 받으면 다음 항목으로 넘어간다. 모든 항목 완료 후 보고서를 생성한다.

#### Post-visit Mode (방문 후)

사용자가 한꺼번에 데이터를 제공하면 (텍스트, 메모, 사진 목록 등) 이를 구조화된 보고서로 정리한다.

### 4. Red Flag Detection

수집된 데이터에서 **위험 요소**를 자동 감지하여 경고한다:

- 전력 부족: TECH-SPEC.md의 필요 전력 > 현장 가용 전력
- 반입 불가: 장비 크기 > 출입문/엘리베이터 크기
- 주변광 과다: 프로젝션인데 암실이 아닌 경우
- 네트워크 없음: 온라인 연결이 필요한데 LAN/WiFi 없음
- 하중 초과: 리깅/LED 하중 > 천장/바닥 한계

### 5. Generate

`templates/SITE-SURVEY.md.tmpl`을 기반으로 마크다운 → `.docx` 변환.

```
{프로젝트 폴더}/SITE-SURVEY.md    ← 원본
{프로젝트 폴더}/SITE-SURVEY.docx  ← Word
```

### 6. Post-generation

```
현장 조사 보고서가 생성되었습니다:
- {프로젝트 폴더}/SITE-SURVEY.docx (Word)
- {프로젝트 폴더}/SITE-SURVEY.md (원본)

⚠️ 주의사항 N건:
- [감지된 위험 요소들 나열]

다음 단계:
- /display-calc 로 현장 치수 기반 디스플레이 계산을 할 수 있습니다
- /install-guide 로 이 현장에 맞는 설치 매뉴얼을 생성할 수 있습니다
- /tech-spec 으로 현장 조건에 맞춰 기술 사양을 보완할 수 있습니다
```

## Data Handoff (입출력 규격)

### Output (쓰는 필드) → 다음 스킬이 읽는 구조
| 출력 필드 | 소비 스킬 | 읽는 방식 |
|----------|----------|----------|
| `## 공간 치수` → 가로, 세로, 천장 높이 | `/display-calc` | 벽면 크기, 투사 거리, 천장 높이 |
| `## 전기` → 차단기 용량(A), 콘센트 수량 | `/tech-spec` | 가용 전력 검증 |
| `## 네트워크` → LAN 포트, WiFi | `/tech-spec` | 네트워크 구성 제약 |
| `## 접근` → 출입문 크기, 엘리베이터, 로딩독 | `/install-guide`, `/tech-spec` | 반입 가능성 검증 |
| `## 환경` → 주변광(lux), 온습도 | `/display-calc` | 밝기 계산 보정 |
| `## Red Flags` → 위험 요소 목록 | `/install-guide` | 설치 시 주의사항 반영 |

## Core Principles

1. **현장 관점** — 사무실이 아닌 현장에서 사용하는 도구다. 모바일 환경에서도 읽기 쉽게.
2. **위험 감지** — 단순 기록이 아닌, 문제를 미리 발견하는 것이 목표.
3. **수치 기반** — 모든 측정값은 단위와 함께 기록한다.
4. **한 번에 하나의 질문** — 현장에서 바쁜 사용자를 압도하지 않는다.
5. **언어를 따른다**
